Indoor Location Services

During my second co-op at Sheridan College, my duties were to research various wireless communication technologies, and determine which technology would be best suited to provide an indoor location service. After conducting some research I recommended using Bluetooth, due to the high availability of low cost Bluetooth beacons. I then began developing the indoor location service on Android, using the Bluetooth beacons as anchor points and trilateration to determine the device's position, as well as researching and recommending other positioning methods we might look into in the future.

Through this project I learned much about how location services work (such as GPS). I also learned about Bluetooth and refined my Android development skills further.

Blog Site

This site is a generified version of a website I wrote for my friend's hobby project. He needed a site which could be hosted free of charge, so I developed this website. It uses GitLab Pages for hosting, Firebase Realtime Database to provide a NoSQL database, and Twitter to provide user authentication. The downside to this of course is that the site is fairly JavaScript heavy, and won't work at all if JavaScript is disabled. Validation is done using database rules in Firebase, thus preventing non-admin users from posting content to the news section. This site is still a work in progress, so there may be bugs here and there.

This project introduced me to Firebase Real Time Database for the first time, and through it I learned a lot about how to implement Firebase in a web project.
